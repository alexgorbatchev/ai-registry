# Bun S3 Object Storage Client Reference (`Bun.S3Client` / `Bun.s3`)

Do not import heavy `@aws-sdk/client-s3` packages or use slow Node.js-based libraries. Bun's native S3 client is written in Zig, supports AWS S3, Cloudflare R2, MinIO, DigitalOcean Spaces, Backblaze B2, and other S3-compatible storage providers, and is highly performant.

---

## 1. Credentials Configuration
`Bun.s3` is a global client singleton that automatically reads environment credentials.
- **Environment variables checked**: `S3_ACCESS_KEY_ID` (or `AWS_ACCESS_KEY_ID`), `S3_SECRET_ACCESS_KEY` (or `AWS_SECRET_ACCESS_KEY`), `S3_REGION`, `S3_ENDPOINT`, `S3_BUCKET`.

To construct a custom client or connect to an alternate service (like Cloudflare R2):
```ts
import { S3Client } from "bun";

const r2 = new S3Client({
  accessKeyId: "my-access-key",
  secretAccessKey: "my-secret-key",
  endpoint: "https://<account-id>.r2.cloudflarestorage.com",
  bucket: "my-bucket",
  virtualHostedStyle: true, // required for hosted-style endpoints
});
```

## 2. Working with S3 Files (`S3File`)
`client.file(key)` returns a lazy file reference (analogous to `Bun.file(path)`). It performs **zero network requests** at creation. Network requests are only made when reading/writing content.

```ts
const s3file = r2.file("documents/report.json");

// 1. Check if file exists
const exists = await s3file.exists();

// 2. Read file contents safely in multiple formats (conforms to Blob interface)
const text = await s3file.text();
const json = await s3file.json();
const bytes = await s3file.bytes(); // Uint8Array
const stream = s3file.stream();     // ReadableStream

// 3. Partial reads (utilizes HTTP Range header)
const headerBytes = await s3file.slice(0, 1024).bytes(); // first 1024 bytes
```

## 3. High-Speed Uploading and Writing
```ts
// Write a string
await s3file.write("file content");

// Write JSON with specific Content-Type metadata
await s3file.write(JSON.stringify({ score: 100 }), {
  type: "application/json",
});

// Stream a Response body directly to S3
const response = await fetch("https://example.com/image.png");
await s3file.write(response, { type: "image/png" });
```

### Streaming Multi-part Uploads (`writer()`)
Bun automatically handles chunking and multipart uploads. You can customize the retries and parallel queue size.
```ts
const writer = s3file.writer({
  retry: 3,                 // auto-retry network errors up to 3 times
  queueSize: 10,             // queue up to 10 upload requests concurrently
  partSize: 5 * 1024 * 1024, // 5MB chunk size
});

writer.write("first chunk\n");
writer.write("second chunk\n");
await writer.flush();       // flush buffers
await writer.end();         // finalize upload
```

## 4. Deleting Files
```ts
await s3file.delete(); // Alias: await s3file.unlink();
```

## 5. Presigning URLs & HTTP Redirection
Generates a signed URL for secure third-party downloads or direct client uploads without exposing S3 credentials.

```ts
// 1. Presign for download (default: GET, expires in 24 hours)
const downloadUrl = s3file.presign({
  expiresIn: 3600, // 1 hour
  contentDisposition: 'attachment; filename="user-report.pdf"',
});

// 2. Presign for upload (PUT)
const uploadUrl = s3file.presign({
  method: "PUT",
  expiresIn: 1800, // 30 minutes
  acl: "public-read",
  type: "image/jpeg",
});
```

### High-performance Redirect Response
To let users download a file without utilizing your server's bandwidth/memory as a proxy, pass the `S3File` directly to a `Response`. Bun automatically issues a `302 Found` response pointing directly to the presigned URL!
```ts
// In your HTTP server fetch handler:
return new Response(s3file); // returns a 302 redirect with the location set to S3!
```

## 6. S3 Static and Global Operations
You can use the static client methods directly to avoid managing file instances.
```ts
import { S3Client } from "bun";

// Check size of file
const sizeInBytes = await S3Client.size("report.pdf", credentials);

// Fetch full metadata (etag, size, lastModified, type)
const stat = await S3Client.stat("report.pdf", credentials);
console.log(stat.etag, stat.lastModified);

// List bucket objects (up to 1,000 keys)
const list = await S3Client.list({ prefix: "uploads/", maxKeys: 100 }, credentials);
for (const obj of list.contents) {
  console.log(obj.key, obj.size);
}
```

## 7. `s3://` Protocol Support
You can pass S3 URLs directly to `fetch` or `Bun.file()` using the `s3://` protocol.
```ts
const res = await fetch("s3://my-bucket/documents/report.json");
const s3File = Bun.file("s3://my-bucket/documents/report.json");
```
